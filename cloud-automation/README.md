# Cloud Automation

Multi-cloud automation and operations across **AWS, Azure, GCP, and Oracle Cloud** —
provisioned as code, monitored, and cost-controlled. Backed by 10+ cloud certifications.

## What I do
- **Provisioning & lifecycle** — infrastructure stood up via IaC and cloud CLIs/APIs, not the console.
- **Hybrid connectivity** — site-to-site and WireGuard tunnels stitching on-prem to cloud.
- **Cost discipline** — right-sized / always-free footprints running real production-style services.

## Best practices I follow
- **Everything as code** — no console snowflakes; landing zones / multi-account from the start.
- **Least-privilege IAM** — scoped roles and tokens, short-lived where possible.
- **Cost at decision time** — tagging + allocation so spend is attributable; right-size continuously.
- **Observability built in** — monitoring, alerting, and status pages alongside the workload.
- **GitOps delivery** — Kubernetes-native where it fits, declarative everywhere.

## Where the field went (dated)
- **2023–2024:** multi-cloud shifts from risk-hedging to a deliberate architecture choice;
  **platform engineering** rises (Internal Developer Platforms treated as products).
- **2024–2025:** **FinOps** goes mainstream — by the 2026 State of FinOps, nearly all practitioners
  manage **AI/GenAI spend** as part of cloud cost (up from ~a third two years earlier).
- **2025–2026:** AI-assisted, context-aware operations and cost optimization become table stakes.

My approach lines up with that: everything as code, observable, and accountable for what it costs to run.
